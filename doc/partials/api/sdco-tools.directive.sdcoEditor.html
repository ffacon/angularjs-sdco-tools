sdcoEditor
directive in module sdco-tools

Description

Defines a new editor container, used in conjunction with sdcoEditorTab

Usage

as element:
<sdco-editor
       compile="{Boolean}"
       compile-js-first="{Boolean}"
       compile-on-demand="{Boolean}"
       read-only
       width="{String}"
       height="{String}"
       js-fiddle="{Boolean}"
       hide-menu="{Boolean}">
</sdco-editor>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
compileBoolean

specify if the editor content can be processed. In such a case, a preview zone will be displayed at top of the editor. HTML tabs will be concatenated and added to the previous zone, CSS tabs will also be applied, and JS content will be executed. The compilation behavior also depends on the compileOnDemand value.

(default: false)

compileJsFirstBoolean

By default, when compile is true, html is processed first, then js is executed. In some specific cases, you can specify that js has to be executed first

(default: false)

compileOnDemandBoolean

if set to true, compilation will be done only when asked (with the play button). Otherwise, it is done each time the editor content changes

(default: false)

readOnlyBoolean

Set the editor tabs in readOnly mode (no modifications allowed)

(default: false)

widthString

a css value to define the editor width

heightString

a css value to define the editor height

jsFiddleBoolean

is the jsFiddle option link is displayed (the user can add it through the editor menu if it is active).

(default: false)

hideMenuBoolean

Hide the top right menu: typically, if set to true, compile to false and jsFiddle to false, the editor is used for display only.

(default: false)
